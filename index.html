Aspen is a dynamic, functional language designed for building scalable and maintainable applications.

Aspen leverages the Erlang VM, known for running low-latency, distributed and fault-tolerant systems, while also being successfully used in web development and the embedded software domain.

To learn more about Aspen, check our getting started guide in the README on Github. Or keep reading to get an overview of the platform, language and tools.

Language Features

Simplicity

Aspen code is a drastically simpler way of generating data for graph databases than writing out Cypher. Sketch out and iterate on data models while spending less energy on Cypher syntax and formatting.

In the small example above, Aspen requires 61% fewer characters to represent the same data, and that ratio will get smaller and smaller the more Aspen there is. Aspen lets you spend less time typing Cypher syntax and more time thinking about your data.

Auto-nicknaming

Writing Cypher directly means having to keep track of which nodes and edges you've instantiated, what their nicknames are, and then building relationships using those nicknames. Aspen relieves you of this mental load, letting you focus just on your data modeling.

(Coming soon) Custom grammars

Aspen aspires to let you write simply but produce rich, complicated graph data. When they're ready, custom grammars will enable you to write simple sentences with no markup, and map the parameters to complex Cypher statements.

Tooling features

Compilation

The Aspen command-line tool compiles Aspen files to Cypher.

In future versions of Aspen, we expect to let you create projects with many Aspen files, which would then build into one big Cypher file.

(Coming soon) Aspen Notebook for live editing

Let's be honest: graph data is complex. Aspen syntax can only simplify to a certain degree, so we have plans to create a live editor. On the left side, you'll write Aspen, and on the right side, you'll see the Cypher and/or the visual graph representation of your data.

You'll be able to use Aspen Notebook to iterate on your data, and if you'd like, publish to a Neo4j instance.
